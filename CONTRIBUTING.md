# Contributing

Thanks for writing something. This guide stays useful only if people keep adding to it.

## The basic idea

The guide has three kinds of content:

1. **Structured responses** — sections where multiple contributors answer the same set of questions in their own voice, attributed. This is how we handle topics where individual variation is the point (grad school applications, fellowships, research paths). You answer the questions for your section; we don't synthesize your voice away.

2. **Synthesized sections with attributed quotes** — sections where the content is more factual and benefits from one clean narrative (course sequencing, the REU landscape, UT-specific logistics). The steward or an editor drafts these from multiple inputs, with direct quotes attributed by name.

3. **Individual essays** — a few sections are just one person's story told their way (falling out of love with math, study abroad narratives, "what I'd do differently"). These stay in the contributor's voice with minimal editing.

Each section's README tells you which format it uses and what's being asked of you.

## How to contribute

1. Pick a section you want to write for. Read its README.
2. Copy `contributors/_template.md` to `contributors/yourname.md` and fill in your bio if you haven't already.
3. For structured-response sections: copy the question template from the section's README into a new file at `sections/XX-section-name/responses/yourname.md` and answer the questions. Some sections have multiple parallel subtopics (e.g., fellowships) and nest responses under a subtopic folder — the section's README is authoritative on where the file goes.
4. For individual essays: write the piece in `sections/XX-section-name/yourname.md`.
5. Open a pull request, or if that's not your thing, send the file to the current steward and they'll add it.

## Style

- **Write in your own voice.** First person. Contractions are fine. If you want to swear, swear. The worst thing you could do is write this like a cover letter.
- **Be specific.** "Take hard classes" is useless; "I took M373K with Professor X in fall of sophomore year and here's what I wish I'd known" is gold.
- **Be honest about what didn't work.** Advice from a success framed as if it was inevitable helps nobody. What did you get wrong? What did you waste time on? What would you do differently?
- **Name names, but kindly.** If a professor was generous with their time, say so. If you had a bad experience, describe the experience rather than attacking the person. Future readers will calibrate.
- **Respect word counts.** They're soft, but they exist so that sections stay readable and contributors don't burn out.

## Attribution and dates

Everything you write will be attributed to you by name, with the date you wrote it. If you'd prefer a pseudonym or initials, say so in your bio file. If at any point you want something edited or removed, email the steward.

## What doesn't belong here

- Anything that outs someone else (e.g., "X struggled with Y") without their explicit permission.
- Specific negative claims about professors or students that you wouldn't say to their face.
- Advice you don't actually believe but feel like you're supposed to say.

## Questions?

Ask the steward. The point of this document is that you shouldn't have to guess.
